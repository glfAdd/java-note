##### 抽象方法

```
1. 抽象方法只声明, 没有具体实现
2. 构造方法，类方法（用 static 修饰的方法）不能声明为抽象方法
3. 必须在抽象类中
4. 子类必须重写父类的抽象方法，或者声明自身为抽象类
5. 抽象方法不能用 private, 需要子类重写
```

##### 抽象类

```
1. 如果一个类包含抽象方法，那么该类必须是抽象类
2. 抽象类除了不能实例化对象之外，类的其它功能依然存在，成员变量、成员方法和构造方法的访问方式和普通类一样
3. 抽象类必须被继承，才能被使用
4. 可以有 0 - n 个抽象方法，以及 0 - n 个具体方法
```
